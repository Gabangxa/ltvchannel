Integration Docs

Everything you need to start capturing UTM attribution in Stripe — in under 5 minutes.

1. Add the JS snippet to your site

Paste this snippet in the <head> of every page on your marketing site (before your checkout redirect). It reads UTM parameters from the URL and stores them in localStorage. 

<script>
(function() {
  var p = new URLSearchParams(window.location.search);
  var keys = ['utm_source','utm_medium','utm_campaign','utm_content','utm_term'];
  keys.forEach(function(k) { if (p.get(k)) localStorage.setItem('ltvc_' + k, p.get(k)); });
})();
</script>

5 lines. No external requests. GDPR-safe (localStorage only, no cookies).

2. Send UTM data to Stripe at checkout

When you create a Stripe Customer or Checkout Session, read the stored UTM values and attach them as Stripe metadata. 

// Node.js example — run server-side at checkout
const utmKeys = ['utm_source','utm_medium','utm_campaign','utm_content','utm_term'];
const utmData = {};
utmKeys.forEach(k => {
  // Pass these from your frontend via a hidden form field or API param
  if (req.body['ltvc_' + k]) utmData[k] = req.body['ltvc_' + k];
});

const customer = await stripe.customers.create({
  email: req.body.email,
  metadata: {
    ltvc_utm_source:   utmData.utm_source   || 'direct',
    ltvc_utm_medium:   utmData.utm_medium   || '(none)',
    ltvc_utm_campaign: utmData.utm_campaign || '(none)',
  }
});

3. Bridge localStorage to your checkout form

Add hidden inputs to your checkout form to pass the stored values server-side. 

<script>
// Add to your checkout page — populates hidden fields automatically
['utm_source','utm_medium','utm_campaign','utm_content','utm_term'].forEach(function(k) {
  var val = localStorage.getItem('ltvc_' + k);
  if (val) {
    var inp = document.getElementById('field_' + k);
    if (inp) inp.value = val;
  }
});
</script>

<!-- Hidden form fields -->
<input type="hidden" id="field_utm_source"   name="ltvc_utm_source" />
<input type="hidden" id="field_utm_medium"   name="ltvc_utm_medium" />
<input type="hidden" id="field_utm_campaign" name="ltvc_utm_campaign" />

Stripe metadata field reference

LTVChannel reads these metadata keys from Stripe Customer objects. All keys are prefixed with ltvc_ to avoid collisions with your existing metadata.

Metadata Key Maps To Example Value Required?
ltvc_utm_source Acquisition channel google, twitter, producthunt Yes — primary grouping key
ltvc_utm_medium Channel medium cpc, organic, referral Recommended
ltvc_utm_campaign Campaign name summer-launch, ph-day1 Optional
ltvc_utm_content Ad variant / link hero-cta, blog-sidebar Optional
ltvc_utm_term Paid search keyword stripe analytics Optional

FAQ

Does this work for customers who arrived before I added the snippet?

Yes — retroactively for customers with existing UTM metadata in Stripe. For customers without metadata, we fall back to grouping them under a direct channel.

Is the JS snippet GDPR-compliant?

It uses localStorage, not cookies, and makes no external network requests. UTM values are considered non-personal data. Consult your DPO for your specific context.

What Stripe permissions do you request?

Read-only access to Customers and Subscriptions. We never request write access, payment method data, or payout account information.

How is LTV calculated?

LTV = sum of all invoice amounts paid by a customer across their entire Stripe lifetime. For active subscriptions we use actual paid revenue, not projected.
